Es un conjunto de personas de diferentes partes del mundo que imparten certificaciones de diferentes areas de la prueba de software.
Se basa en certificar lo que la empresa busca especificamente.
Es recomendado que se tome el curso para gente que esta por egresar de areas de prubea de software, profesionista y personas con deseo de especializarse en el perfil.
Al ejecutor de pruebas se le asigna un proyecto, debe revisar  y validar los casos de prueba, revis los datos de la prueba y el plan de ejecucion, espera la señal del lider para empezar con las pruebas
Realiza los casos de prueba, documenta los resultados y graba las evidencias y las almacenas, si existe algun defecto se envia al area de desarrollo y si hay dudas se revisan con el lider de pruebas.

Que es una prueba de software?
Es la validacion y verificacion de los requerimientos funcionales del software.
Tambien es el analisis la calidad del sw como el uso de recursos,el desempeño etc.

Existen diferentes tipos de pruebas: Funcional y No funciona.
La funcional se divide en la forma manual se revisa que el sistema haga lo que se requiere y el automatizado es programado en lenguajes y permite automatizar las acciones manuela es que un usuario realiza.
Las no funcionales: se dividen en el desempeño que esta enfocado en el uso del CPU mientras se realizan las pruebas, entre otras cosas. En la seguridad se utiliza para buscar vulnerabilidades del sistema en servicios web.

El ciclo de vida de un proyecto de prueba.

-Analisis
Se valida la factibilidad del proyecto. 
-Planeacion
Se crea un plan de trabajo para identificar como se va a trabajar el proyecto.
-Diseño
Se crean los diseños del sw y de los casos de prueba.
-Ambiente
Se trabaja para que este listo el proyecto para la ejecucion.
-Ejecucion
Se ejecutan los casos de prueba.
-UAT (Pruebas de aceptacion de usuario)
El usuario ejecuta pruebas mas especificas.
-Cierre
Se entrega el proyecto de forma adecuada.

Que es un caso de prueba?
Es la validacion de que el sistema haga lo que debe hacer y no haga lo que no debe hacer.

Script/Procedimientos de pruebas.

Es el conjunto de pasos para reproducion un caso de prueba.
Es el medio para validar un caso de pruebas.
No debe contener dato duros.
Deber tener los pasos para ejecutar la prueba.
Contiene una descripcion de paso, un resultado esperado y un estado de paso.

Datos de prueba.
Son aquellos datos que se requieren para ejecutrar una prueba.
Se divide en estaticos y dinamicos.

TestWare.

Proceso de negocio tienen Requerimientos tiene escenarios, casos de pruebas, proc
edimiento de prueba, conjunto de casos de prueba.
Set de pruebas: Es un conjunto de casos de prueba relacionados que tiene caracteristicas similares.
El objetivo es integrar los ciclos para su ejecucion.

Que es la ejecucion de pruebas?
Es tomar un caso de prueba previamente diseñado por un tester y seguir con el procedimiento de prueba.
Si el resultado esperado es igual al actual es caso es pasado de lo contrario se levanta un defecto al desarrollador.
Siempre se documenta una evidencia de la ejecucion de la prueba.

La ejecucion de pruebas de detona entre la fase de diseño y la fase de UAT.

Que es un Testset?
Un conjunto de casos de prueba con caracteristicas similares.

Ciclo de pruebas:

Es un conjunto de Set de pruebas para poder ejecutarlos.
Se ejecuta el 100% de los casos programados.
El lider de pruebas es el responsabla de armar el Set de pruebas.


Estatus de un caso de prueba:

No ejecutado: el ejecutor no existe ninguna accion en este caso de prueba.
Pasado: El caso esperado es igual al actual que arrojo el sistema en la ejecucion.
Fallado: El resultado esperado no es igual al actual.
Detenido o bloqueado: Debido a un caso de prueba fallado este caso no se puede ejecutar.


Tipos de proyectos de pruebas:

Nuevos: Se desarrolla un proyecto de sw desde 0% puede tomar desde 3 meses a un año segun los requerimientos.
Mantenimientos: Modificaciones y actualizaciones a proyectos existentes. Toma semana.
Incidentes: Problemas prodcutivos. Toma dias 

Clasificacion de proyectos:

Evolutivos son para operar la empresa, Normativos son regulaciones de entidades externas y Administrativos son para gestiones de la misma empresa
